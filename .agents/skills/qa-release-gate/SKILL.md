# QA Release Gate

이 스킬은 구현이 끝난 뒤 실제 출시 가능 상태인지 판단하는 절차를 제공한다.

## QA 체크 순서

1. 관련 테스트 명령을 실행한다.
2. 성공 경로와 실패 경로를 확인한다.
3. API 응답 shape와 UI 표시를 비교한다.
4. `_workspace/05_qa_report.md`에 근거를 남긴다.
5. 배포 메모와 환경변수를 `_workspace/06_release_checklist.md`에 정리한다.

## QA 보고서에 반드시 포함할 것

- 재현 절차
- 실행 명령
- 통과/실패 결과
- 남은 위험
- 수정 필요 여부

## 배포 게이트 체크리스트

- 테스트 통과
- 린트 통과
- 빌드 통과
- 필수 환경변수 정의
- 롤백 포인트 확인
- 배포 후 확인 URL 또는 경로

## 이 저장소의 기본 명령

- `npm test`
- `npm run lint`
- `npm run build`

필요한 경우 기능 범위에 맞는 테스트 파일만 먼저 좁혀서 실행한 뒤, 마지막에 전체 게이트를 다시 확인한다.
