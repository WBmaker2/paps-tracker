# Design Handoff

프론트엔드 구현자가 바로 화면을 만들 수 있게 디자인 핸드오프를 정리한다.

## 출력 파일

- `_workspace/02_design_handoff.md`

## 포함 항목

1. 목표 사용자와 핵심 작업
2. 페이지/화면 목록
3. 각 화면의 주요 섹션
4. 폼 입력, 버튼 액션, 오류 상태
5. 빈 상태, 로딩 상태, 성공 상태
6. 구현 메모

## 작성 원칙

- 시각 스타일보다 흐름과 상태를 우선한다.
- 컴포넌트 경계를 암시할 정도로 구조를 내려 쓴다.
- 프론트와 백엔드가 모두 읽을 수 있게 입력/출력 포인트를 명시한다.

## 최소 템플릿

```markdown
# 기능 이름
## 사용자 목표
## 화면 목록
## 화면별 구조
## 상태 정의
## 문구/액션 메모
## 구현 리스크
```
