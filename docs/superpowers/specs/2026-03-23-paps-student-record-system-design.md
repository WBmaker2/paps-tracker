# PAPS 학생 기록 시스템 설계

작성일: 2026-03-23

## 목적

학생이 체력 측정 후 스스로 기록을 입력하고, 교사는 학교/학급/학생/세션을 관리하며, 기록은 연결된 구글 스프레드시트에 저장되는 PAPS 학생 기록 시스템을 만든다.

이 시스템은 다음 두 가지 목적을 함께 만족해야 한다.

1. 공식 PAPS 평가 기록을 안전하게 수집하고 관리한다.
2. 월 1회 수준의 연습 기록을 누적해 학생 개인의 향상도를 표와 그래프로 제공한다.

## 문서 기반 운영 원칙

참고 문서:

- `경기도 4세대 나이스 학생건강체력평가(PAPS) 매뉴얼(저용량).pdf`
- `2025학년도 단위학교 학생 체력증진 기본 계획(알림용).pdf`
- `2025 PAPS 관련 Q_A 자료.pdf`
- `[별표 3] 신체능력검사 항목 및 방법(제7조제5항 관련)(학교건강검사규칙).hwp`
- `[별표 4] 신체능력검사(필수평가) 기준표(제8조제2항 관련)(학교건강검사규칙).hwp`
- `2026학년도 학생 건강체력평가(PAPS) 실시 계획.hwp`

시스템은 아래 원칙을 따라야 한다.

- 학년별, 종목별, 필요 시 성별 기준이 다르므로 단일 기준으로 일괄 처리하지 않는다.
- 점수보다 `원측정값`과 `단위`를 우선 저장한다.
- 공식 평가의 경우 학년/성별/종목 기준표에 따라 자동 등급을 계산할 수 있어야 한다.
- 연습 기록에서는 등급을 보여주지 않고 원측정값과 향상도만 보여준다.
- 미실시 종목은 입력 대상에서 제외하거나 입력을 막아야 한다.
- 측정 방식은 종목별 표준 절차를 따라야 하며, 시간/거리/회수/단위 규칙을 보존해야 한다.
- 교사가 세션을 열어 두었을 때만 학생 입력이 가능해야 한다.

## 사용자

### 교사

- 구글 로그인으로 관리 페이지에 접속한다.
- 학교 정보, 학급, 학생 명단, 세션, 종목, 구글 시트 연결을 관리한다.
- 공식 평가 세션과 연습 세션을 만든다.
- 학생이 제출한 여러 시도 중 대표값을 선택한다.
- 결과 표, 그래프, 동기화 상태를 확인한다.

### 학생

- 로그인 없이 교사가 연 세션에 접속한다.
- 이름만 선택하고 기록을 입력한다.
- 제출 직후에만 자기 누적표와 자기 종목별 그래프를 잠깐 확인할 수 있다.
- 이후에는 다시 조회하지 않는다.

## 핵심 시나리오

### 1. 교사 관리

교사는 관리 페이지에서 다음을 수행한다.

- 학교 정보 등록
- 학년/반 관리
- 학생 명단 등록 및 수정
- 학교가 사용할 종목 선택
- 구글 시트 링크 연결
- 기록 세션 생성 및 열기/닫기

### 2. 학생 기록 입력

학생 입력 흐름:

1. 교사가 연 세션 링크 또는 QR로 진입
2. 이름 선택
3. 현재 세션의 종목 기록 입력
4. 제출
5. 제출 직후 자기 결과와 간단한 향상도 확인
6. 다음 학생으로 전환

학생은 학년, 반, 종목, 공식/연습 여부를 직접 선택하지 않는다. 이 값들은 모두 세션에 의해 결정된다.

### 3. 교사 대표값 확정

- 학생이 같은 세션에서 여러 번 입력할 수 있다.
- 모든 입력 시도는 누적 저장된다.
- 교사는 시도 목록을 보고 최종 대표값을 선택한다.
- 표, 그래프, 공식 요약, 시트 요약에는 교사가 선택한 대표값을 사용한다.

## 제품 접근안

선택된 접근안은 `세션 기반 학교 운영형(B안)`이다.

이 접근안의 특징:

- 교사가 세션 단위로 운영을 통제한다.
- 학생은 이름 선택만으로 매우 단순하게 입력한다.
- 공식 기록과 연습 기록은 세션 유형으로 구분한다.
- 한 반 또는 두 반 동시 기록을 지원한다.
- 학생과 교사가 각각 필요한 수준의 결과를 본다.

## 화면 구조

### 교사 화면

주요 메뉴:

- 학교 설정
- 학생 명단
- 세션 관리
- 결과 조회
- 시트 연동

교사는 세션 생성 시 아래 항목을 설정한다.

- 학년도
- 학년
- 대상 반: 1개 또는 2개
- 세션 유형: `공식 평가` 또는 `연습 기록`
- 종목
- 입력 화면 유형: `1반형` 또는 `2반 분할형`
- 세션 열림/닫힘 상태

### 학생 화면

학생 화면은 최대한 단순해야 한다.

- 진입
- 이름 선택
- 기록 입력
- 제출
- 즉시 결과 확인

### 2반 분할형

- 좌우 분할 화면으로 두 반을 동시에 기록한다.
- 두 반은 반드시 `같은 종목`을 동시에 기록한다.
- 이 설정은 교사 대시보드에서만 바꿀 수 있다.

## 세션 구조

세션은 학생 입력을 제어하는 가장 중요한 단위다.

세션 속성:

- 세션명
- 학년도
- 학년
- 대상 반 목록
- 종목
- 세션 유형
- 입력 화면 유형
- 열림/닫힘 상태
- 생성 교사
- 생성 시각

세션 규칙:

- 학생은 열린 세션에서만 입력할 수 있다.
- 학생은 세션 내에서 이름만 선택한다.
- 세션이 닫히면 추가 입력은 불가능하다.
- 학생 입력 화면에서 공식/연습 여부를 선택하지 않는다.

## 공식 평가와 연습 기록

### 공식 평가

- 학년/성별/종목 기준표를 이용해 자동 등급 계산 가능
- 미실시 종목은 입력 차단
- 교사가 대표값을 확정한 뒤 공식 요약에 반영

### 연습 기록

- 학교가 종목을 선택해 운영 가능
- 등급은 보여주지 않음
- 원측정값과 향상도 중심으로 제공

## 입력 및 기록 규칙

- 학생은 이름만 선택한다.
- 학생이 같은 세션에서 여러 번 제출해도 모두 저장한다.
- 각 입력은 `시도 순번`을 가진다.
- 대표값은 자동 선택하지 않고 교사가 선택한다.
- 공식 통계와 그래프는 대표값 기준으로 계산한다.
- 원본 시도 기록은 항상 보존한다.

## 학생 결과 노출 규칙

- 학생은 제출 직후에만 자기 누적표와 자기 종목별 그래프를 볼 수 있다.
- 학생은 다른 학생의 기록을 볼 수 없어야 한다.
- 공용 기기 운영을 고려해 학생 조회 화면은 다음 학생 전환 시 바로 종료된다.

## 구글 시트 연동 설계

### 기본 방향

- 하나의 스프레드시트 안에 여러 탭으로 관리한다.
- 모든 원본 기록은 누적 보존한다.
- 요약 탭은 앱이 계산해 작성한다.

### 탭 구조

- `설정`
- `학생명단`
- `세션기록`
- `학생요약`
- `공식평가요약`
- `오류로그`
- 필요 시 `수정로그`

### 탭 역할

`설정`

- 학교 정보
- 연결 정보
- 사용 종목
- 학년/반 설정

`학생명단`

- 학년
- 반
- 번호
- 이름
- 성별

`세션기록`

- 모든 학생 입력 기록을 세로 누적 저장
- 시도별 원본을 모두 보존

`학생요약`

- 학생별 최신 대표값
- 최고 기록
- 종목별 변화량

`공식평가요약`

- 공식 평가 세션의 대표값과 등급 중심 요약

`오류로그`

- 시트 주소 오류
- 쓰기 실패
- 잘못된 종목 매핑
- 재동기화 이력

## 구글 시트 링크 처리

- 교사 관리 페이지에 구글 시트 링크 입력 기능 제공
- 링크 형식을 먼저 검사한다.
- 유효한 링크면 시트 ID를 추출한다.
- 주소가 잘못되었거나 형식이 맞지 않으면 즉시 오류 메시지를 보여준다.
- `구글 시트 양식 생성` 버튼은 템플릿 복사 링크를 제공한다.
- 사용자는 복사한 자기 시트 링크를 앱에 붙여넣어 연결한다.

## 시트 저장 실패 처리

선택된 정책:

- 시트 저장이 실패해도 학생 입력은 먼저 앱에 저장한다.
- 교사 화면에는 `동기화 실패` 경고를 보여준다.
- 오류 내역은 `오류로그` 탭에 남긴다.
- 교사는 `재동기화` 기능으로 다시 시도할 수 있다.

이 정책으로 수업 흐름을 끊지 않으면서 데이터 유실 위험을 줄인다.

## 관리 규칙과 제약

- 관리 페이지는 교사 구글 로그인 사용자만 접근 가능해야 한다.
- 학생은 로그인하지 않는다.
- 학생 입력은 반드시 열린 세션에 종속되어야 한다.
- 학생이 접하는 화면에서는 학교/반/종목/공식 여부를 다시 선택하지 않는다.
- 교사가 명단을 수정할 수 있어야 한다.
- 학교가 연습 기록 종목을 선택할 수 있어야 한다.

## 보관 정책

- 학생 기록, 대표값 선택 이력, 동기화 오류 로그는 최소 해당 학년도 동안 보관한다.
- 다음 학년도 시작 전에는 학교가 이전 학년도 데이터를 조회 전용으로 유지할지, 별도 시트로 백업할지 선택할 수 있어야 한다.
- 오류로그와 수정로그는 원본 추적을 위해 삭제보다 보관을 우선한다.
- 구글 시트 구조상 학년도 구분 컬럼을 필수로 두어 연도별 분리가 가능해야 한다.

## 향상도 표시

학생과 교사 모두 종목별 향상도를 확인할 수 있어야 한다.

학생:

- 제출 직후 짧은 결과와 간단 그래프만 노출

교사:

- 학생별 누적표
- 종목별 그래프
- 공식/연습 구분 조회
- 대표값 선택 후 반영 결과 확인

## 범위 밖

이번 설계에서 아직 포함하지 않는 항목:

- 학생 개인 장기 로그인
- 학부모용 별도 뷰
- 다학교 통합 관리자
- 3개 반 이상 동시 분할 입력
- 등급 기준표를 교사가 직접 편집하는 기능

## 구현 시 주의사항

- 공용 기기 사용을 고려해 학생 화면은 큰 버튼과 적은 단계로 구성한다.
- 입력값 검증은 종목과 단위 기준으로 수행한다.
- 공식 등급 계산은 기준표 데이터와 분리해 유지보수 가능하게 설계한다.
- 시트 연동 오류가 있어도 입력 손실이 없도록 비동기 재시도 구조를 둔다.
- 그래프와 표는 대표값 기준과 원본 기록 기준을 혼동하지 않도록 명확히 구분한다.

## 승인된 사용자 결정 사항

- 학생 입력 방식은 `이름 선택만`
- 학생은 `공용 기기 1대`에서 순서대로 입력
- 교사가 연 세션에서만 입력 가능
- 학생은 제출 직후에만 자기 누적표와 자기 그래프 확인 가능
- 공식 평가와 연습 기록은 학생이 아니라 교사 세션 설정으로 구분
- 월별 연습 기록 종목은 학교가 선택 가능
- 제품 접근안은 `세션 기반 학교 운영형(B안)`
- 화면 구조 섹션 승인
- 구글 시트는 `하나의 스프레드시트 안에 여러 탭으로 관리`
- 2반 분할형은 가능하고, 두 반은 `같은 종목`을 동시에 기록
- 모든 시도 기록을 저장하고, 대표값은 `교사 선택값`
- 연습 기록에서는 등급을 보여주지 않음
- 시트 저장 실패 시 앱에 먼저 저장하고 나중에 재동기화
