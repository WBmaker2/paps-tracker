# 디자인 핸드오프

## 기능명

- 교사 결과 화면 필터 및 검색 패널

## 사용자 목표

- 교사가 결과 화면에서 특정 학생 또는 특정 조건의 기록을 빠르게 찾는다.
- 필터를 적용한 상태에서도 대표 기록 선택과 결과 검토 흐름을 끊지 않는다.
- 화면이 복잡해지더라도 기존 핵심 기능인 `대표값 선택`, `요약 재계산`, `요약 다운로드`가 더 찾기 쉬워진다.

## 대상 화면

- `/teacher/results`

이 기능은 새로운 페이지를 만드는 대신, 기존 결과 화면의 정보 구조를 재배치하는 방식으로 진행한다.

## 화면 구조

### 1. 상단 헤더

- 기존 `AppShell` 헤더는 유지한다.
- 제목과 설명은 그대로 사용한다.
- 네비게이션도 현재 구조를 유지한다.

### 2. 왼쪽 메인 컬럼

왼쪽 컬럼은 다음 순서로 배치한다.

1. `필터 및 검색 패널`
2. `결과 개수/적용 상태 요약`
3. `시도 기록 목록`

#### 2-1. 필터 및 검색 패널

- 둥근 카드 형태
- 결과 테이블 바로 위에 위치
- 내부 구성:
  - 학생명 검색 입력
  - 학년 선택
  - 반 선택
  - 종목 선택
  - 세션 유형 선택
  - `필터 초기화` 버튼

#### 2-2. 결과 개수/적용 상태 요약

- 필터 카드 하단 또는 결과 테이블 상단에 배치
- 예시 문구:
  - `현재 8건 / 전체 26건`
  - `공식평가 · 5학년 · 1반 · 앉아윗몸앞으로굽히기`
- 활성 필터는 작은 pill 형태로 보인다.

#### 2-3. 시도 기록 목록

- 현재 `ResultTable` 카드 구조를 유지한다.
- 단, 목록 상단 제목은 `시도 기록`에서 다음처럼 확장한다.
  - 제목: `시도 기록`
  - 보조 문구: `학생별 시도 기록을 검색하고, 조건을 좁혀 대표 기록을 확정합니다.`

### 3. 오른쪽 보조 컬럼

오른쪽 컬럼은 다음 구성을 유지한다.

1. `선택 학생 추이 그래프`
2. `동기화 상태 카드`
3. `구글 시트 반영 현황`
4. `요약 내려받기와 결과 미리보기`

단, 왼쪽 목록에 필터가 적용되면 오른쪽 1, 2번은 `현재 필터 결과 안에서 선택된 학생` 기준으로 동작한다.

## 입력 요소 정의

### 학생명 검색

- 형태: 텍스트 입력
- placeholder:
  - `학생 이름으로 검색`
- 동작:
  - 이름 일부 포함 검색
  - 대소문자 구분 없음
  - 입력 즉시 반영

### 학년 선택

- 형태: 드롭다운
- 기본값:
  - `전체 학년`
- 옵션:
  - 현재 결과 목록에서 계산 가능한 학년만 노출

### 반 선택

- 형태: 드롭다운
- 기본값:
  - `전체 반`
- 동작:
  - 학년이 선택되면 해당 학년 반만 노출
  - 현재 선택 반이 유효하지 않으면 자동 초기화

### 종목 선택

- 형태: 드롭다운
- 기본값:
  - `전체 종목`

### 세션 유형 선택

- 형태: 2개 pill 버튼 또는 segmented control
- 옵션:
  - `전체`
  - `공식`
  - `연습`

## 상태 정의

### 기본 상태

- 필터 미적용
- 전체 결과 목록 표시
- 오른쪽 그래프는 첫 번째 유효 학생 또는 현재 포커스 학생 기준

### 필터 적용 상태

- 결과 개수와 활성 필터 pill 노출
- 결과 목록은 조건에 맞는 row만 보여준다
- 오른쪽 그래프와 동기화 카드도 필터된 목록 안의 포커스 row 기준으로 바뀐다

### 빈 결과 상태

- 전체 데이터는 있지만 현재 필터 결과가 0건인 경우
- 결과 목록 자리에는 빈 상태 카드 표시
- 문구:
  - 제목: `조건에 맞는 측정 결과가 없습니다.`
  - 설명: `검색어를 지우거나 필터를 초기화하면 전체 결과를 다시 볼 수 있습니다.`
- 액션:
  - `필터 초기화` 버튼

### 전체 빈 상태

- 전체 결과 자체가 아직 없는 경우
- 기존 빈 상태 흐름 유지

### 대표값 선택 성공 상태

- 현재 `대표값이 업데이트되었습니다.` 피드백 유지
- 필터 상태는 유지한다
- 선택된 row가 필터 범위 안에 있으면 그대로 남는다

### 대표값 선택 실패 상태

- 현재 에러 문구 출력 유지
- 필터 입력값은 유지한다

## 포커스 규칙

- 현재 선택된 row가 필터 적용 후에도 목록에 남아 있으면 그대로 유지한다.
- 현재 선택된 row가 필터로 제외되면, 필터 결과의 첫 번째 row를 새 포커스로 잡는다.
- 필터 결과가 0건이면:
  - 그래프 카드: `조건에 맞는 학생이 없어 그래프를 표시할 수 없습니다.`
  - 동기화 카드: 숨기거나 비활성 상태로 전환

## 요약 미리보기와 다운로드 처리

- `학생요약 미리보기`, `공식평가요약 미리보기`, `CSV/XLSX 다운로드` 섹션은 이번 단계에서 `전체 시트 기준`으로 유지한다.
- 대신 섹션 상단에 작은 안내 문구를 추가한다.
  - `이 요약표는 현재 화면 필터와 별개로 전체 연결 시트 기준입니다.`

이렇게 하면 결과 목록 필터 기능과 시트 요약 기준이 다르다는 점을 교사가 오해하지 않게 된다.

## 모바일 배치

- 모바일에서는 필터 패널이 결과 목록 위에 세로로 쌓인다.
- 검색 입력은 한 줄 전체 폭 사용
- 나머지 필터는 2열 또는 1열 반응형 배치
- 오른쪽 컬럼 카드들은 결과 목록 아래로 내려간다

## 문구/액션 메모

### 카드 제목

- 필터 카드 제목: `검색 및 필터`
- 필터 카드 설명: `학년, 반, 종목, 세션 유형으로 결과를 빠르게 좁혀 볼 수 있습니다.`

### 버튼 문구

- `필터 초기화`
- `1회차 대표값으로 선택`
- `2회차 대표값`

### 보조 문구

- `현재 8건 / 전체 26건`
- `학생요약과 공식평가요약은 전체 연결 시트 기준으로 표시됩니다.`

## 구현 메모

### 권장 구현 방식

- 1차 구현은 `클라이언트 필터링`으로 진행한다.
- 이유:
  - 현재 결과 페이지는 이미 필요한 row 목록을 서버에서 한 번 구성하고 있다.
  - 필터는 표시 로직에 가깝고, 별도 API round trip 없이도 즉시 반응하는 편이 교사 화면에 더 자연스럽다.

### 권장 컴포넌트 분리

- 새 클라이언트 래퍼 컴포넌트 예시:
  - `TeacherResultsWorkspace`
- 하위 컴포넌트 예시:
  - `ResultsFilterPanel`
  - `ResultTable`
  - `TeacherProgressChart`
  - `SyncStatusCard`
  - `SummaryExportsCard`

### 프론트엔드가 필요로 하는 row 메타데이터

현재 결과 row에 다음 정보가 안정적으로 있으면 좋다.

- `gradeLabel` 또는 `grade`
- `classId`
- `eventId`
- `sessionType`
- 검색용 정규화 이름

이 정보가 없다면 다음 단계인 API 계약 문서에서 shape를 정리한다.

### 상태 관리 원칙

- 필터 상태는 결과 페이지 내부 client state로 관리한다.
- 대표값 선택 후에는 목록 전체를 다시 fetch하지 않고, 현재 row 상태만 갱신하는 방향을 우선한다.
- 필터와 포커스 학생 선택 state는 분리한다.

## 구현 리스크

- 현재 페이지가 `activeSession` 중심이므로, 필터 범위를 `현재 세션 내부`로 볼지 `로드된 전체 결과`로 볼지 계약 단계에서 명확히 해야 한다.
- 학년/세션 유형 정보가 현재 row에 직접 없으면 프론트에서 파싱하지 말고 서버에서 명시적으로 내려주는 편이 안전하다.
- 필터 패널이 너무 커지면 결과 테이블보다 조작 UI가 더 눈에 띄게 될 수 있어, 카드 밀도 조절이 필요하다.
